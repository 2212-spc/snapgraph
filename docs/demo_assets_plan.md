# Demo Assets Plan

The project should ship a small visual set before public release.

## Required Screens

- Recall: question input, user-stated reasons, answer.
- Evidence Paths: source -> saved reason -> project/open loop cards.
- Memory: source browser with user-stated and AI-inferred status.
- Report: corpus health, review paths, cognitive gaps.
- Settings: provider state, key env name, workspace health.

## Capture Flow

1. Start `snapgraph demo` with the demo corpus.
2. Use MockLLM for deterministic screenshots.
3. Capture desktop and mobile viewports.
4. Store final assets under `docs/assets/`.
5. Reference assets from README after they are generated.

Do not include API keys or private source material in screenshots.
